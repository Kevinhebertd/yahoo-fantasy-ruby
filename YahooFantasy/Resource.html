Overview

Provides consistent and (what I believe to be) consistent request context for the Yahoo Fantasy API documented at: developer.yahoo.com/fantasysports/guide/

Note that in terms of use, Resources and Collections (although documented separately) should not provide that much separate logic in the grand scheme of things. The YahooFantasy::Resource#api method is left available in order to provide your own custom request logic.
